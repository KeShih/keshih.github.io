More algorithms on perfectly balanced photo gallery


Recently there was an article by Johannes Treitz submitted to Hacker News about how to display a set of pictures as nicely as possible. The article doesn't have a formal description of the problem, so here is my take on what it mean by perfectly balanced.

Problem1

Given a sequence of nnn rectangles where the iiith rectangle has width WiW_iWi​ and height HiH_iHi​. Also given are width WWW and target height HHH. Partition the sequence to kkk consecutive subsequences, such that we scale the rectangles keeping the aspect ratio, and each subsequence of rectangles fills the entire width WWW, and is close to the target height HHH.

The problem is not so well defined. To make sure the heights are close, do we minimize the sum of all differences? Minimize the maximum difference? Or maybe, we just want to minimize the difference between consecutive rows, such that the first row has height close to hhh.

Nevertheless, in real applications, the exact definition doesn't matter that much. Treitz reduce the problem to the linear partition problem.

Problem2

Let a1,…,ana_1,\ldots,a_na1​,…,an​ be a sequence of positive reals. We want to partition it into kkk consecutive subsequences, such that the maximum sum over each subsequence is minimized. Formally, find k+1k+1k+1 positions b1=1,b2,…,bk,bk+1=nb_1=1,b_2,\ldots,b_{k},b_{k+1}=nb1​=1,b2​,…,bk​,bk+1​=n, such that max⁡i=1k∑j=bibi+1aj\max_{i=1}^{k} \sum_{j=b_i}^{b_{i+1}} a_jmaxi=1k​∑j=bi​bi+1​​aj​ is minimized.

ai=Wi/Hia_i = W_i/H_iai​=Wi​/Hi​ is the aspect ratio. This article will explore the techniques to solve the problem in O(kn)O(kn)O(kn) time. It is only a high level overview, and leaves the details unfilled.

1 Minimize total difference

Let's first consider a simpler problem for demonstration.

Problem3

Let a1,…,ana_1,\ldots,a_na1​,…,an​ be a sequence of positive reals. We want to partition it into kkk consecutive subsequences, such that the total difference between the sum of each consecutive sequence and the average sum of the whole sequence is minimized. Formally, let μ=1k∑i=1nai\mu = \frac{1}{k}\sum_{i=1}^n a_iμ=k1​∑i=1n​ai​. Find k+1k+1k+1 positions b1=1,b2,…,bk,bk+1=nb_1=1,b_2,\ldots,b_k,b_{k+1}=nb1​=1,b2​,…,bk​,bk+1​=n, such that ∑i=1k∣∑j=bibi+1aj−μ∣\sum_{i=1}^{k} |\sum_{j=b_i}^{b_{i+1}} a_j - \mu|∑i=1k​∣∑j=bi​bi+1​​aj​−μ∣ is minimized.

We will solve this problem with a reduction to a problem on a DAG, and then apply dynamic programming. Since we can always turn a problem that ask us to find the cost to a problem that ask us to find a construction that achieve the cost(with the same time/space bound). We will only concern with the cost version of the problem as it's much clearer. (One can implement some arrows in Haskell to make DP for construction as easy as DP for cost, sounds like a nice project. )

Build a directed graph D(V,A)D(V,A)D(V,A), where V={v1,…,vn}V=\set{v_1,\ldots,v_n}V={v1​,…,vn​}, (vi,vj)∈A(v_i,v_j)\in A(vi​,vj​)∈A if and only if i≤ji\leq ji≤j. (We can make it i<ji < ji<j instead, depend on if we consider a empty sequence a valid sequence.)

We have a weight function w(i,j)w(i,j)w(i,j) that assign weights to arc (vi,vj)(v_i,v_j)(vi​,vj​). Define w(i,j)=∣∑k=ij−1ai−μ∣w(i,j) = |\sum_{k=i}^{j-1} a_i - \mu|w(i,j)=∣∑k=ij−1​ai​−μ∣.

If we find a kkk-edge path of minimum weight from v1v_1v1​ to vnv_nvn​, then this implies a solution to Problem 3.

2 Solve the minimum weight kkk-edge path problem

Define C(d,i)C(d,i)C(d,i) to be the ddd-edge path from v1v_1v1​ to viv_ivi​ with minimum weight. We want to find C(k,n)C(k,n)C(k,n).

If we set w(j,i)=∞w(j,i)=\inftyw(j,i)=∞ if j>ij>ij>i, then we have a better representation.

C(d,i)=min⁡1≤j≤nC(d−1,j)+w(j,i)\displaystyle C(d,i) = \min_{1\leq j\leq n} {C(d-1,j) + w(j,i)} C(d,i)=1≤j≤nmin​C(d−1,j)+w(j,i)

There are knknkn entries in the DP table for CCC, and C(d,i)C(d,i)C(d,i) requires O(n)O(n)O(n) time to compute. This means the algorithm will take O(kn2)O(kn^2)O(kn2) time.

3 Improve the time complexity

There is a standard technique on totally monotone matrices that can reduce the complexity of the problem to O(kn)O(kn)O(kn).

Definition4

A weight function www is Monge if for every 1<i+1<j≤n1<i+1<j\leq n1<i+1<j≤n, we have w(i,j)+w(i+1,j+1)≤w(i,j+1)+w(i+1,j)\displaystyle w(i,j) + w(i+1,j+1)\leq w(i,j+1) + w(i+1,j) w(i,j)+w(i+1,j+1)≤w(i,j+1)+w(i+1,j) .

Theorem5

www is Monge.

Proof

Let ∑k=i+1j−1ai−μ=m\sum_{k=i+1}^{j-1} a_i - \mu = m∑k=i+1j−1​ai​−μ=m w(i,j)+w(i+1,j+1)=∣∑k=ij−1ai−μ∣+∣∑k=i+1jai−μ∣=∣ai+m∣+∣aj+m∣≤∣ai+aj+m∣+∣m∣=w(i,j+1)+w(i+1,j)\displaystyle \begin{aligned} w(i,j)+w(i+1,j+1) &= |\sum_{k=i}^{j-1} a_i - \mu| + |\sum_{k=i+1}^{j} a_i - \mu|\\ &= |a_i + m| + |a_j + m|\\ &\leq |a_i+a_j+m| + |m|\\ &= w(i,j+1)+w(i+1,j) \end{aligned}w(i,j)+w(i+1,j+1)​=∣k=i∑j−1​ai​−μ∣+∣k=i+1∑j​ai​−μ∣=∣ai​+m∣+∣aj​+m∣≤∣ai​+aj​+m∣+∣m∣=w(i,j+1)+w(i+1,j)​ To prove the ≤\leq≤, see that ai,aja_i,a_jai​,aj​ are positive, one can consider either mmm is negative or positive, and notice either way the inequality holds true.

Define a matrix MdM^dMd, such that Mj,id=C(d−1,j)+w(j,i)M_{j,i}^d = C(d-1,j) + w(j,i)Mj,id​=C(d−1,j)+w(j,i), the original recurrence become C(d,i)=min⁡1≤j≤nMj,id\displaystyle C(d,i) = \min_{1\leq j\leq n} {M^d_{j,i}} C(d,i)=1≤j≤nmin​Mj,id​ In other words, C(d,i)C(d,i)C(d,i) is the iiith column's minima of MdM^dMd.

Theorem7

If www is Monge, then MdM^dMd is a n×nn\times nn×n totally monotone matrix.

Using the SMAWK algorithm, all column minimas can be found in O(n)O(n)O(n). Finding C(d,i)C(d,i)C(d,i) takes only O(1)O(1)O(1) time on average!

Here is the very simple code to show how this can be done easily if we have a Haskell implementation of the SMAWK algorithm. The indexing is a bit different from the description in the article.

import Data.Array
import SMAWK

minCostkEdgePath k n w = d!(k-1,n)
  where
    d = array ((0,0),(k-1,n)) [ ((i,j), f i j) | i<-[0..k-1],j<-[0..n]]
    f 0 i = w 0 i
    f k i = m k (p!(k,i)) i
    p = array ((1,0),(k,n)) [((z,i),t) |z<-[1..k],(i,t)<-zip [0..n] (columnMinima (m z) (n+1) (n+1))]
    m k j i = (+) (d!(k-1,j)) (w j i)

minCostMuPartition k xs = minCostkEdgePath k n w
    where 
        w i j 
         | i <= j    = abs $! s!j - s!i - avg
         | otherwise = 2*m
        s   = listArray (0, n-1) $ scanl (+) 0 xs
        n   = length xs
        m   = sum xs
        avg = m/fromIntegral k

4 Solve the linear partition problem

Now, returning to the original problem. Again, we can reduce the problem to a problem on a directed graph. This time, w(i,j)=∑k=ij−1aiw(i,j) = \sum_{k=i}^{j-1} a_iw(i,j)=∑k=ij−1​ai​. The weight of a path is the maximum weight of the edges in the path. A kkk-edge path with minimum weight implies the solution to the original problem.

C(d,i)=min⁡1≤j≤nMj,id\displaystyle C(d,i) = \min_{1\leq j\leq n} M^d_{j,i} C(d,i)=1≤j≤nmin​Mj,id​

where Mj,id=max⁡(C(d−1,j),w(j,i))M^d_{j,i} = \max (C(d-1,j),w(j,i))Mj,id​=max(C(d−1,j),w(j,i)).

Just like Problem 3, this describes a O(kn2)O(kn^2)O(kn2) time algorithm. Compare w,Cw,Cw,C and MdM^dMd with the previous problem to see there isn't much difference.

5 Improve the time complexity, again

Can we define an alternative to the Monge property? Yes, we can extend this to algebraic Monge property. Just replace +++ with some associative operation ⊕\oplus⊕, ≤\leq≤ with a total order that is ordered with respect to ⊕\oplus⊕, i.e. a≤a⊕ba \leq a\oplus ba≤a⊕b.

So it seems, using the algorithm above with little modification, we can solve the problem in O(kn)O(kn)O(kn) time because we can just replace +++ by min⁡\minmin.

However, algebraic Monge property in general doesn't imply totally monotone matrix.

Consider the simple matrix, and our operation is max⁡\maxmax.

[1202]\begin{bmatrix} 1& 2\\ 0& 2 \end{bmatrix}[10​22​]

max⁡(1,2)=max⁡(0,2)\max(1,2)=\max(0,2)max(1,2)=max(0,2), but the matrix is not totally monotone. 1>01>01>0 but 2>22\not > 22​>2.

If instead the operation is strictly compatible, i.e. a⊕b<a⊕ca \oplus b< a \oplus ca⊕b<a⊕c if b<cb < cb<c, then we can always produce a totally monotone matrix. This is not the case with max⁡\maxmax.

Some readers who are familiar with LpL^pLp space might point out for any sequence of reals a1,…,ana_1,\ldots,a_na1​,…,an​ and ϵ>0\epsilon>0ϵ>0, there exist a p≥1p\geq 1p≥1, such that (∑i=1n∣ai∣p)1/p−max⁡i=1n∣ai∣<ϵ(\sum_{i=1}^n |a_i|^p)^{1/p} - \max_{i=1}^n |a_i|< \epsilon(∑i=1n​∣ai​∣p)1/p−maxi=1n​∣ai​∣<ϵ. So if we don't need to be exact, pick a large enough ppp as an exponential is good enough. This opens up a can of numerical analysis.

A more restrictive but good enough variant of the Monge property hold.

Definition8

www has the strict bottleneck Monge property if either of the following is true for 1<i+1<j1<i+1<j1<i+1<j:

  1. max⁡(w(i,j),w(i+1,j+1))<max⁡(w(i+1,j),w(i,j+1))\max(w(i,j), w(i+1,j+1)) < \max(w(i+1,j),w(i,j+1))max(w(i,j),w(i+1,j+1))<max(w(i+1,j),w(i,j+1)).
  2. max⁡(w(i,j),w(i+1,j+1))=max⁡(w(i+1,j),w(i,j+1))\max(w(i,j), w(i+1,j+1)) = \max(w(i+1,j),w(i,j+1))max(w(i,j),w(i+1,j+1))=max(w(i+1,j),w(i,j+1)) and min⁡(w(i,j),w(i+1,j+1))≤min⁡(w(i+1,j),w(i,j+1))\min(w(i,j), w(i+1,j+1)) \leq \min(w(i+1,j),w(i,j+1))min(w(i,j),w(i+1,j+1))≤min(w(i+1,j),w(i,j+1)).

Our www in consideration has strict bottleneck Monge property. Because all the numbers are positive, w(i,j+1)>w(i+1,j+1)>w(i+1,j)w(i,j+1)>w(i+1,j+1)>w(i+1,j)w(i,j+1)>w(i+1,j+1)>w(i+1,j), w(i,j+1)>w(i,j)>w(i+1,j)w(i,j+1)>w(i,j)>w(i+1,j)w(i,j+1)>w(i,j)>w(i+1,j). A simple case check on the relation between w(i,j)w(i,j)w(i,j) and w(i+1,j+1)w(i+1,j+1)w(i+1,j+1) will give the desired proof. Why this property? You can check by a case by case proof that this property implies total monotonicity.

Theorem9

If www has the strict bottleneck Monge property, then MdM^dMd is a n×nn\times nn×n totally monotone matrix.

There seems to be a direct proof by analyze 12 different cases, but I got too bored after the second case. One can read [1] for the proof. It doesn't contain the exact theorem, but a result that implies this one. The proof is quite involved. Their idea is to construct a new matrix with a new operation over sorted sequences of numbers instead of just numbers. Prove this matrix is strictly compatible and has the algebraic Monge property, and show this can always be done if the original matrix has the strict bottleneck Monge property.

6 Rethink the original problem

We have developed a O(kn)O(kn)O(kn) algorithm for Problem 2. As we can see from the practical application, this is a O(n2)O(n^2)O(n2) algorithm because the kkk is of the order nnn.

For a few hundred photos, we can afford to run it in real time. This will reach a limit once there are thousand of photos.

Can we do better? Yes, by considering a different kind of reduction(thank god we didn't formally define what close means). Instead of compute kkk and try to fit kkk rows, why not just make sure each row's width is almost the width we want and scale accordingly? We want each rows to approximately have width WWW, we don't really care how many rows there are.

Problem11

Let a1,…,ana_1,\ldots,a_na1​,…,an​ be a sequence of positive reals, and a number μ=W/H\mu = W/Hμ=W/H. We want to partition it into consecutive subsequences, such that the maximum difference between the sum of each consecutive sequence and the μ\muμ is minimized. Formally, find a kkk and a sequence of k+1k+1k+1 numbers b1=1,b2,…,bk,bk+1=nb_1=1,b_2,\ldots,b_{k},b_{k+1}=nb1​=1,b2​,…,bk​,bk+1​=n, such that max⁡i=1k∣∑j=bibi+1aj−μ∣\max_{i=1}^{k} |\sum_{j=b_i}^{b_{i+1}} a_j - \mu|maxi=1k​∣∑j=bi​bi+1​​aj​−μ∣ is minimized.

This is the exact same problem described on section 5 of [1]. It can be solve in O(n)O(n)O(n) time.
