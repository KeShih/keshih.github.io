Reconstructing edge-disjoint paths, a tighter analysis


This is a small improvement of the analysis of [1], this assume you already read that paper.

0.0.0.0.0.1

Given a simple undirected graph of nnn vertices and mmm edges, construct a data structure such that we can query two vertices uuu and vvv, and it returns the maximum edge disjoint paths between uuu and vvv.

We let MF(n,m)MF(n,m)MF(n,m) to be the time to compute ststst-maximum flow on a undirected unit capacity graph.

The current state of art in the article is:

  1. Proprocessing time O(nMF(n,m))O(nMF(n,m))O(nMF(n,m)).
  2. O(n3)O(n^3)O(n3) space.
  3. O(α(n)λ(u,v)n)O(\alpha(n)\lambda(u,v)n)O(α(n)λ(u,v)n) query time.

We can improve this by having a tighter analysis.

First, if fff and ggg be kkk-edge disjoint path from xxx to yyy and yyy to zzz respectively. Let mmm be the number of edges in f∪gf\cup gf∪g, then there exist an algorithm to output kkk-edge disjoint path from xxx to zzz in O(m)O(m)O(m) time. One can see this by update section 222 of their paper by using a stable matching algorithm without the dummy edges, because this is just stable matching with incomplete lists[2].

Second, acyclic flow of value fff uses only O(fn)O(\sqrt{f}n)O(f​n) edges(Theorem 3.1 in [3]), therefore we do not have to store any non-acyclic flows. This also speed up all computations because the number of edges are smaller.

Third, we can use O(n2.5log⁡n)O(n^{2.5}\log n)O(n2.5logn) space and get a query time of O(λ(u,v)n)O(\sqrt{\lambda(u,v)}n)O(λ(u,v)​n). In fact there is a time space trade off [4].

Thus finally, we get

  1. Proprocessing time O(nMF(n,m))O(nMF(n,m))O(nMF(n,m)).
  2. O(n2.5log⁡n)O(n^{2.5}\log n)O(n2.5logn) space.
  3. O(λ(u,v)n)O(\sqrt{\lambda(u,v)}n)O(λ(u,v)​n) query time.

With even better analysis, we can obtain the following [5].

  1. Proprocessing time O(nMF(n,m))O(nMF(n,m))O(nMF(n,m)).
  2. O(mn3/2log⁡n)O(\sqrt{m}n^{3/2}\log n)O(m​n3/2logn) space.
  3. O(kn)O(\sqrt{k}n)O(k​n) query time for a kkk edge disjoint path from uuu to vvv.
