Minimum cuts with restrictions


There is a nice algorithm design technique for cuts: fix a small number of partial solutions, and guess all possible solutions built from the partial solution. Here are two demonstrations.

1 kkk-size-cut

A cut (X,Xˉ)(X,\bar{X})(X,Xˉ) such that both ∣X∣,∣Xˉ∣≥k|X|,|\bar{X}|\geq k∣X∣,∣Xˉ∣≥k is called a kkk-size-cut.

Problem1

Given a graph GGG, find a minimum kkk-size-cut.

The standard algorithm is based on finding all possible min-STSTST-cut, where S,TS,TS,T are all possible disjoint sets of size kkk. The running time is O(n2k)O(n^{2k})O(n2k) min-cut computations.

One can improve it by first fix an arbitrary set of kkk vertices YYY. Consider a min-kkk-size-cut (X,Xˉ)(X,\bar{X})(X,Xˉ). Let S′=X∩YS'=X\cap YS′=X∩Y and T′=Xˉ∩YT'=\bar{X}\cap YT′=Xˉ∩Y. We can try to find the min-cut of all possible SSS and TTT such that S′⊂SS'\subset SS′⊂S,T′⊂TT'\subset TT′⊂T and ∣S∣=∣T∣=k|S|=|T|=k∣S∣=∣T∣=k. Since we don't know what S′S'S′ and T′T'T′ are, we will try all 2k2^k2k possibilities. Here the S′S'S′ and T′T'T′ are the partial solutions. This gives us an algorithm with running time O(2knk)O(2^k n^k)O(2knk) min-cut computations. The idea is also used in computing matroid connectivity.

A further improvement depending on the fact that there are only O(nk−1)O(n^{k-1})O(nk−1) cuts we try to avoid. We can enumerate all the cuts from smallest to largest, with a delay of a single application of Hao-Orlin algorithm [1]. The running time of Hao-Orlin algorithm is approximately a single maximum flow. One of the smallest (nk−1)+1{n \choose k-1}+1(k−1n​)+1 cuts is the min-kkk-size-cut. The running time is O(nk−1)O(n^{k-1})O(nk−1) Hao-Orlin computation [2].

It's interesting to wonder if the running time can be improved, especially for the case where k=2k=2k=2. Fix a set S={s,t}S=\{s,t\}S={s,t} of size 222. There are two cases, either the min-222-size-cut crosses SSS, then one of the the 333 smallest ststst-cuts is our solution. The other case is SSS is on one side of the min-222-size-cut, and we are interested in finding a cut so the side doesn't contain SSS has at least 222 vertices.

This prompt a interesting problem:

Problem2

Find the second smallest ststst-cut, if we already have a min-ststst-cut and it's corresponding flow(or some other useful information obtained through a push relabel flow computation).

Can we solve this in O(m)O(m)O(m) time? What if we also know the min-ststst-cut is induced by ttt?

2 222-restricted-cut

A problem which appeared as a question on cstheory.

Problem3

Given a graph GGG, find the minimum cut under the constraint that each side is connected and has at least 222 vertices. (Assume it exists).

This is the kkk-restricted edge connectivity problem when k=2k=2k=2.

λk(G)\lambda_k(G)λk​(G), the kkk-restricted edge connectivity of GGG, defined as the smallest number of edges such that the removal result exactly 222 connected component, each with at least kkk vertices. The rest of the article describes the algorithm by Esfahanian and Hakimi [3].

λ2(G)\lambda_2(G)λ2​(G) can be found in O(m2)O(m^2)O(m2) flow computations. The idea is to contract any two independent edges eee and e′e'e′ to sss and ttt, and then find a ststst-min-cut. The cut will give us the desired partition.

It can be improved with the idea of fixing a partial solution. Consider a single edge eee that incident to a vertex with lowest degree, contract it to vertex sss. Pick another edge e′e'e′ that not incident to sss, we contract it to ttt. The min-cut between sss and ttt reflects a 222-restricted cut. If eee is on one side of the min-222-restricted cut, then this algorithm finds it in O(m)O(m)O(m) flow computations by trying all possible e′e'e′.

Otherwise, eee is an edge crossing every min-222-restricted cut. Let e=uve=uve=uv and and wlog deg⁡(u)=δ\deg(u)=\deltadeg(u)=δ, the min degree. We fix another partial solutions where uuu and vvv are on different side of the min-222-restricted cut. One can contract any edge incident to uuu and any edge incident to vvv and apply a flow computation. There are at most deg⁡(u)deg⁡(v)≤δn=O(m)\deg(u) \deg(v)\leq \delta n = O(m)deg(u)deg(v)≤δn=O(m) flow computations.
