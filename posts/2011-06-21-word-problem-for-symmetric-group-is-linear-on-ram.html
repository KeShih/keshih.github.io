Word problem for symmetric group is linear on RAM


1 Linear time algorithm for symmetric group

Problem1The word problem for symmetric groups

Input: www is a word of length lll from the presentation Sn=⟨x1,x2,…,xn∣xi2=1,xi+1xixi+1=xixi+1xi,xixj=xjxi⟩S_n = \langle x_1,x_2,\ldots,x_n \mid x_i^2 = 1, x_{i+1}x_ix_{i+1} = x_ix_{i+1}x_i, x_ix_j = x_jx_i \rangleSn​=⟨x1​,x2​,…,xn​∣xi2​=1,xi+1​xi​xi+1​=xi​xi+1​xi​,xi​xj​=xj​xi​⟩ where ∣i−j∣≠1|i-j|\neq 1∣i−j∣​=1.

Output: Return true if www is the identity, else return false.

The representation was crucial for coming up with a linear time algorithm respect to nnn and lll. This is not a word problem on one group, but on a set of group.

Theorem2

The following is a O(n+l)O(n+l)O(n+l) algorithm for the word problem for symmetric groups on RAM.

  1. Produce an array a of size nnn, Such that a[i] = i. (Array start with index 1)
  2. Reading the word letter by letter. If one encounters xix_ixi​, swap(a[i],a[i+1]).
  3. Test if the a[i] == i for all iii. If true, return true, else return false.
Proof

The algorithm takes O(n+l)O(n+l)O(n+l) time is obvious. The correctness needs to be justified.

xix_ixi​ can be represented as the transposition (i i+1)(i~i+1)(i i+1). Define (n n+1)=(n 1)(n~n+1) = (n~1)(n n+1)=(n 1).

Represent a element of the group as a permutation π\piπ in the 2 line notation. wlog, assume π(j)=i\pi(j) = iπ(j)=i and π(k)=i+1\pi(k) = i+1π(k)=i+1.

(1⋯j⋯k⋯nπ(1)⋯i⋯i+1⋯π(n))(i i+1)=(1⋯j⋯k⋯nπ(1)⋯i+1⋯i⋯π(n)) \begin{pmatrix} 1 & \cdots &j & \cdots & k& \cdots & n \\ \pi(1) & \cdots & i & \cdots & i+1 & \cdots & \pi(n)\end{pmatrix}(i~i+1) = \begin{pmatrix} 1 & \cdots &j & \cdots & k& \cdots & n \\ \pi(1) & \cdots & i+1 & \cdots & i & \cdots & \pi(n)\end{pmatrix} (1π(1)​⋯⋯​ji​⋯⋯​ki+1​⋯⋯​nπ(n)​)(i i+1)=(1π(1)​⋯⋯​ji+1​⋯⋯​ki​⋯⋯​nπ(n)​)

If we call jjj the index of iii if π(j)=i\pi(j) = iπ(j)=i. Then each transposition is a swap of indices.

The value of a[i] in the algorithm stores the index of iii. Array a represent the identity iff a[i] = i for all iii.

This proves the the correctness of the algorithm.

The algorithm can be modified so it runs in O(ℓn!)O(\ell n!)O(ℓn!) time for a Turing machine. For each fixed nnn, a Turing machine MMM can construct another Turing machine MnM_nMn​, such that it store the state of the array as the state of the Turing machine.

This proves every symmetric group is automatic. For any fixed SnS_nSn​, the Turing machine MnM_nMn​ can solve the problem in O(ℓ)O(\ell)O(ℓ) time without writing anything to the tape and can only move to the right, which is equivalent to a finite state automata.

Remark

Automatic is a property of a group, not a set of groups. That’s why nnn is ignored in the O(ℓn!)O(\ell n!)O(ℓn!), because it’s fixed for each SnS_nSn​. I was confused for a while before I read a concrete definition.

One can construct a purely symbolic algorithm that apply only the group relations. We measure the time by the amount of group relations used.

Siegel proposed an O(l2)O(l^2)O(l2) algorithm to solve this problem.
