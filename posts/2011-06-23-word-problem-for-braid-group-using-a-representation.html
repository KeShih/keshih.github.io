Word problem for braid group using a representation


Problem1The word problem for braid groups

Input: www is a word of length lll from the presentation ⟨σ1,σ2,…,σn−1∣σi+1σiσi+1=σiσi+1σi,σiσj=σjσi⟩\langle \sigma_1,\sigma_2,\ldots,\sigma_{n-1} \mid \sigma_{i+1}\sigma_i\sigma_{i+1} = \sigma_i\sigma_{i+1}\sigma_i, \sigma_i\sigma_j = \sigma_j\sigma_i \rangle⟨σ1​,σ2​,…,σn−1​∣σi+1​σi​σi+1​=σi​σi+1​σi​,σi​σj​=σj​σi​⟩ where ∣i−j∣≠1|i-j|\neq 1∣i−j∣​=1.

The word problem for braid group was solved a long time ago with Artin combing. It requires one to express pure braid group PnP_nPn​ as a semidirect product of free groups F1,…,Fn−1F_1,\ldots,F_{n-1}F1​,…,Fn−1​. It is slow and quite difficult, at least I wasn’t able to come up with an explicit algorithm for it.

Another way was to create a faithful homomorphism ∗:Bn→Aut(Fn)^*: B_n \to \mathop{\mathrm{Aut}}(F_n)∗:Bn​→Aut(Fn​). This is what I implemented in Haskell.

If Bn→Aut(Fn)B_n \to \mathop{\mathrm{Aut}}(F_n)Bn​→Aut(Fn​) is faithful and it sends www to w∗w^*w∗, then www is III iff w∗(a)=aw^*(a) = aw∗(a)=a for every generator aaa of FnF_nFn​.

In the a survey by Jonathan Boiser, there is one explicate map. Where tit_iti​ are generators of FnF_nFn​, and σi\sigma_iσi​ are the generators of the braid group. The inverse can easily be found.

Using a recursive definition. Let any word of the form (σiw)∗(\sigma_i w)^*(σi​w)∗ be σi∗∘w∗\sigma_i^* \circ w^*σi∗​∘w∗. The algorithm is obvious. Test if w∗(ti)=tiw^*(t_i) = t_iw∗(ti​)=ti​ for every generator in FnF_nFn​ is applying a list of σi\sigma_iσi​ to elements in FnF_nFn​.

I wrote the program in Haskell, and posted it on github.

The analysis: Given a word with length lll in BnB_nBn​, how long does it take to solve the problem with this algorithm? Each application of σi∗(u)\sigma_i^*(u)σi∗​(u) to some word uuu takes O(∣u∣)O(|u|)O(∣u∣) time. One then free reduce. σi\sigma_iσi​ is applied lll times.

An application of σi∗\sigma_i^*σi∗​ can potentially triple the length of the word. If one shows that it can only increase the length of the word by a constant term(given one started from a generator), then we have a O(l2n)O(l^2 n)O(l2n) algorithm.

Braid groups are automatic. If this natural algorithm solves the problem in O(l2n)O(l^2 n)O(l2n) time, it is not a surprise. However, it is likely not true. The running time is more likely to be O(cln)O(c^l n)O(cln) for some constant ccc. Although I can prove neither σi∗\sigma_i^*σi∗​ and increase the length by a constant factor or by a constant. The best known algorithm in 2000 was provided by a paper of Hessam Hamidi-Tehrani. It runs in O(l2n+lnlog⁡n)O(l^2 n + l n \log n)O(l2n+lnlogn) time, and was proved with advanced techniques.
