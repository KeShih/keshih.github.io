Bounded regression on data streams


1 Bounded Regression on Data Streams

Hsien-Chih sent me this problem. Similar problem has been asked on Quora. He noticed it might be solved in near linear time using min-cost circulation. Here we show a generalization.

3 min-cost circulation on series-parallel graphs

The constructed graph is a two terminal series-parallel graph. There is a simple procedure to solve min-cost flow problem on series-parallel graphs. Consider a series connection of two edges, each with cost function fff and ggg. We can replace it with an edge with cost function f+gf + gf+g. If it is a parallel connection, then we can replace it with one edge and a cost function f □ gf~\square~gf □ g, where □\square□ is the infimal convolution: (f □ g)(x)=inf⁡yf(x−y)+g(y)(f~\square~g)(x)= \inf_y f(x-y) + g(y)(f □ g)(x)=infy​f(x−y)+g(y).

Once we have a good data structure to represent the costs, we can reduce the graph to one single edge easily, and find the minimum cost circulation. In particular, if the cost are continuous, convex and piecewise linear in a interval and ∞\infty∞ everywhere else, and the total number of breakpoints is nnn, then Booth and Tarjan has an algorithm that runs in O(nlog⁡n)O(n\log n)O(nlogn) time [1].

Because all edge has a cost function with at most 111 breakpoint. The bounded regression problem can be solved in O(nlog⁡n)O(n\log n)O(nlogn) time.

We can express all the problems as min-cost circulation problem on a appropriate graph. If the min-cost circulation algorithm on those graphs have the same running time as current best algorithm, it would imply something more general is acting in the background.

Here is what we know.

  1. L1L_1L1​ error: This post shows it can be solved in O(nlog⁡n)O(n\log n)O(nlogn) time using the min-cost circulation formulation. It matches the running time of specialized algorithms.
  2. L2L_2L2​ error: It can be solved in O(n)O(n)O(n) time, but doesn’t come from the quadratic cost min-cost circulation formulation.
  3. L∞L_\inftyL∞​ error: It can be solved in O(n)O(n)O(n) time. However, it doesn’t come from the minimax circulation problem. (In the minimax circulation, the cost is the largest edge cost incurred by the circulation).
  4. L0L_0L0​ error: It can be solved in O(nlog⁡n)O(n\log n)O(nlogn) time. This is equivalent to the longest non-decreasing subsequence problem.

This prompt the following two natural problems:

  1. Can min-cost circulation with quadratic cost on series parallel graph have O(n)O(n)O(n) time solution? This is in fact possible when all edges have no capacity[3]. But with capacity, even for a edge with a lower bound of 000 and 000 cost, we don’t know.

  2. What about minimax circulation? We can’t find any study of minimax circulation on series-parallel graphs.
