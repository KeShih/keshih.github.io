Rectangles in point set


Problem1

Given nnn points in the plane PPP, find if any 444 of them are the vertices of some axis-aligned rectangle.

The idea is we guess the left vertical segment of the rectangle, and see what would be the right vertical segment. If we pick an left vertical line lll, and then for each (x,y)∈l∩P(x,y)\in l\cap P(x,y)∈l∩P, we consider all the points with the same yyy coordinate and to the right of xxx, and add a counter to the vertical lines that contains it. This can be done in linear time with respect to number of vertices with the same yyy coordinate if one already builds a data structure before hand. If any counter become 222, then we are done.

It is a O(n2)O(n^2)O(n2) time algorithm, since the time it takes is at most O(n)O(n)O(n) to guess each left vertical segment. One could, however analyze this algorithm better, and realize the time is actually O(ln)O(ln)O(ln), where lll is the maximum number of points on a horizontal line. Indeed, we look at a horizontal line that contain lll points, and realize for each point on the horizontal line, we need to increment a counter at most lll times.

We can also solve the problem in O(t2+n)O(t^2+n)O(t2+n) time, where ttt is the number of vertical lines. Note since we can stop as soon as a counter become 222. The number of counter we will increase is at most t2t^2t2. Indeed, the counter depend only one two vertical lines.

There is an O(n3/2)O(n^{3/2})O(n3/2) algorithm [1], using this O(n2)O(n^2)O(n2) algorithm as subroutine.

We consider the set of vertical lines that contains at least 222 points. This give us two set of vertical lines, SSS are the lines with at most n\sqrt{n}n​ points and LLL are the lines contain at least n+1\sqrt{n}+1n​+1 lines.

There are 3 possibilities. There is a rectangle with two sides contained in SSS, or one side in SSS one in LLL, or both in LLL.

For the first case, just use our O(ln)O(ln)O(ln) algorithm, which gives us O(n3/2)O(n^{3/2})O(n3/2) time.

For the second case, consider we pick one line in SSS and the union of all the points in the lines in LLL. For each point in SSS, we will find if this point is in some line in LLL, if it is, we increment a counter for that line. We would increment at most n\sqrt{n}n​ counters. The running time is therefore ∑l∈S∣l∩P∣(log⁡n+n)=O(n3/2)\sum_{l\in S} |l\cap P|(\log n + \sqrt{n}) = O(n^{3/2})∑l∈S​∣l∩P∣(logn+n​)=O(n3/2).

Once we are done with the first two case, consider remove all the points lying on the small lines. Now we only have large lines. Since there are at most n\sqrt{n}n​ large lines, we can rotate the plane and run the algorithm again, but this time, we know all the lines are small.

As noted in [2], rectangle finding problem can be reduced to finding a C4C_4C4​ in a bipartite graph. The vertices are X={x∣(x,y)∈P}X=\set{x|(x,y)\in P}X={x∣(x,y)∈P} and Y={y∣(x,y)∈P}Y=\set{y|(x,y)\in P}Y={y∣(x,y)∈P}, and the edges are PPP. So this is a nice interaction with graph theory. In the same article, some stronger bounds were established. First, if the graph have degeneracy ddd, the running is O(md)O(md)O(md). They used the fact and lead to finding a C4C_4C4​ takes O(m4/3)O(m^{4/3})O(m4/3) time [2].

We briefly mention how this is done.

First, it is known that if m≥cn1/2m \geq c n^{1/2}m≥cn1/2, then there exists a C4C_4C4​. If d≥cn3/2d\geq c n^{3/2}d≥cn3/2, then one can find a subgraph that has degree at least ddd in O(m)O(m)O(m) time. Note this implies in the subgraph, there exists a C4C_4C4​, which there is a particular algorithm to solve for it.
