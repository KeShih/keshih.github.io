L1L_1L1​ linear regression


I read an article on the errors in visualization. The example of forcing a relationship by cherry-picking scales is delightful. I recommend reading it.

I am interested in how to mislead people as much as possible, while being absolutely correct. The article inspires the following problem. Given 2 vectors x,y∈Rn\bm{x},\bm{y}\in \R^nx,y∈Rn. Let 1\bm{1}1 be the all 111 vector in Rn\R^nRn. We are interested in finding a,b∈Ra,b\in \Ra,b∈R, such that we minimizes ∥y−(ax+b1)∥p\|\bm{y}-(a\bm{x}+b\bm{1})\|_p∥y−(ax+b1)∥p​. Here ppp is either 1,21,21,2 or ∞\infty∞.

Note the problem is precisely the same as the linear regression problem. In the linear regression problem, we are given points (x1,y1),…,(xn,yn)(x_1,y_1),\ldots,(x_n,y_n)(x1​,y1​),…,(xn​,yn​) and we are interested in find a line f(x)=ax+bf(x) = ax+bf(x)=ax+b, such that it minimizes the error, defined as

∑i=1n∥yi−f(xi)∥p\displaystyle \sum_{i=1}^n \|y_i - f(x_i)\|_p i=1∑n​∥yi​−f(xi​)∥p​

For p=2p=2p=2, there is a O(n)O(n)O(n) time algorithm because there is a closed formula. For p=∞p=\inftyp=∞, the problem can be rewritten as a linear program with 333 variables and nnn constraints. Using Megiddo's result [1], there is a O(n)O(n)O(n) time algorithm to solve this problem.

Strangely, I could not find just how hard is it to solve the problem when p=1p=1p=1. This case is called the least absolute deviations. Statisticians just don't care about worst case running time as CS people do.

There are a few methods I found. One is to write it as a linear program on n+2n+2n+2 variables and nnn constraints and solve it using the simplex method. There are a bunch of other algorithms that specializes the simplex algorithm on this particular problem.

One can realize in the optimal solution, it is a line that goes through two of the points. The native algorithm can try all possible O(n2)O(n^2)O(n2) lines. For each line, the algorithm can compute the error in O(n)O(n)O(n) time. Together it is a O(n3)O(n^3)O(n3) time algorithm. There is a smarter algorithm. The optimal line that contains the point can actually be found in O(n)O(n)O(n) time. Indeed, consider the line passes through the point (x,y)(x,y)(x,y). We consider changing the slope of the line, while maintaining it still contain (x,y)(x,y)(x,y). One can see a minimum will be reached at some line. Indeed, assume we reorder the points, so yi−yxi−x≤yi+1−yxi+1−x\frac{y_i-y}{x_i-x}\leq \frac{y_{i+1}-y}{x_{i+1}-x}xi​−xyi​−y​≤xi+1​−xyi+1​−y​ (namely, increasing slope). Let kkk be the smallest value such that the sum of ∑i=1k∣xi−x∣≥∑i=k+1n∣xi−x∣\sum_{i=1}^k |x_i-x|\geq \sum_{i=k+1}^n |x_i-x|∑i=1k​∣xi​−x∣≥∑i=k+1n​∣xi​−x∣ indicate the point (xk,yk)(x_k,y_k)(xk​,yk​). The line determined by (x,y)(x,y)(x,y) and (xk,yk)(x_k,y_k)(xk​,yk​) is the desired line. This can be computed in linear time by finding weighted median. Hence one can show the running time is O(n2)O(n^2)O(n2). This is the idea of [2]. As far as I know, this seems to be the state of the art. There are many iterative methods, which of course depends on the input values.

After discussing with Qizheng He, he suggested the following approach.

Given a line f(x)=ax+bf(x)=ax+bf(x)=ax+b, can one compute the error quickly? It is possible to decompose it to few halfspace counting queries (allowing weights). In halfspace counting queries problem, we are given nnn points with weights, we can preprocess it and obtain a data structure. Each query to a data structure is a halfspace, the output is the sum of all elements in the halfspace. In 222D, there exists a preprocessing time O~(n4/3)\tilde{O}(n^{4/3})O~(n4/3) and query time O~(n1/3)\tilde{O}(n^{1/3})O~(n1/3) data structure [3]. Let S+S^+S+ be the set of points above fff, and S−S^-S− be the set of points below fff. The result is precisely the following.

∑(x,y)∈S+y−ax−b+∑(x,y)∈S−ax+b−y\displaystyle \sum_{(x,y)\in S^+} y - ax-b + \sum_{(x,y)\in S^-} ax+b - y (x,y)∈S+∑​y−ax−b+(x,y)∈S−∑​ax+b−y

Let's consider the second part, ∑(x,y)∈S−ax+b−y=a∑(x,y)∈S−x+∣S−∣b−∑(x,y)∈S−y\sum_{(x,y)\in S^-} ax+b - y = a\sum_{(x,y)\in S^-}x + |S^-|b -\sum_{(x,y)\in S^-}y∑(x,y)∈S−​ax+b−y=a∑(x,y)∈S−​x+∣S−∣b−∑(x,y)∈S−​y. Note the 333 terms can each be solved with a halfspace counting query, consider all points lies below fff. This shows in 666 halfspace counting queries.

Consider the function gp(s)g_p(s)gp​(s) defined for a point ppp to be the error for the line with slope sss that contains ppp. The function is bitonic, therefore we can do a ternary search to find the minimum. The only problem is that we do not have a space of the slopes for ppp that we can quickly go through.

Halfspace counting data structure is built on top of partition tree, which uses the following to partition the points by putting them into simplices.

Theorem1

Let SSS be a set of nnn points on the plane. We can find O(r)O(r)O(r) triangles, such that each one contains Θ(n/r)\Theta(n/r)Θ(n/r) elements in SSS, and every line intersects O(r)O(\sqrt{r})O(r​) triangles.

So we can look into partition tree directly. Say the partition tree has a branching factor of rrr, then we can spend O~(r)\tilde{O}(r)O~(r) time to generate one testing line for each of the rrr partitions. We can do ternary search on those lines in O(log⁡r)O(\log r)O(logr) error computations. Once we find a few adjacent lines. Together it would cut O(r1/2)O(r^{1/2})O(r1/2) partitions each with O(nr)O(\frac{n}{r})O(rn​) elements. Now we just have to recurse. I did not completely verify. But setting rrr to be n1/3n^{1/3}n1/3 is fine. This would imply a O~(n4/3)\tilde{O}(n^{4/3})O~(n4/3) running time algorithm.
