Induced subgraph with constant diameter

Is this the best possible for general GTkG_T^kGTk​ too? Not at all. To show this, we introduce an algorithmic problem.

Problem3

Preprocess a rooted tree TTT where the elements in the tree TTT has weights from a semigroup (S,+)(S,+)(S,+), such that we can answer the following query in kkk semigroup operations.

Query Input: xxx and yyy where xxx is an ancestor of yyy.

Query Output: The sum of weights of the unique simple path between xxx and yyy.

Alon and Schieber showed this problem can be solved by precompute O(nλ(k,n))O(n\lambda(k,n))O(nλ(k,n)) path sums [???]. Where λ(k,n)\lambda(k,n)λ(k,n) is related to the inverse Ackermann function, and λ(4,n)=O(nlog⁡∗n)\lambda(4,n)=O(n\log^*n)λ(4,n)=O(nlog∗n).

Theorem4

If TTT has nnn vertices, then GTkG_T^kGTk​ have O(nλ(k,n))O(n \lambda(k,n))O(nλ(k,n)) edges.

Proof

A simple reduction from our tree problem by input our rooted tree TTT into Alon and Schieber's algorithm. For each path sum they preprocess, we create an edge in our GGG. The diameter of all GvG_vGv​ is at most 2k2k2k, since for any 2 vertices xxx and yyy, we find it's lowest common ancestor www in TTT, then there is a path of length kkk from xxx to www and another of length at most kkk from www to yyy.

Their paper argued an lower bound of preprocess Ω(nλk(n))\Omega(n\lambda_k(n))Ω(nλk​(n)) path sums for a single rooted path. In our model, we can add O(n)O(n)O(n) edges is sufficient. An edge from the leaf to every other vertex, making the diameter only 222. This is because we have a lot more freedom in our diameter problem.

In their problem, their path sum is actually divide the path into kkk subpaths, our path of length 222 would imply there is an overlap in the sum.

One might to think the reason we can do it in O(n)O(n)O(n) edges comes directly from in the undirected model we can traverse back from the leaf and allow overlaps. Maybe the following problem might give us an lower bound.

Problem5

Preprocess a rooted tree TTT where the elements in the tree TTT has weights from a semigroup (S,+)(S,+)(S,+), such that we can answer the following query in kkk semigroup operations.

Query Input: xxx and yyy where xxx is an ancestor of yyy and yyy is a leaf.

Query Output: The sum of weights of any walk lies strictly in between xyxyxy and that start at xxx and ends at yyy.

Unfortunately, this does not give us an lower bound. Following the proof in Alon's and Schieber's paper, we can show any algorithm would need to precompute at least O(nλk(n))O(n\lambda_k(n))O(nλk​(n)) paths in order to output the solution using kkk semigroup operations on this particular tree: TTT is a rooted tree consist of one single path of length nnn, and for each vertex, we add one single dangling leaf.

Again, for this graph, we can add at most nnn edges and force diameter to be at most 222. Take the lowest leaf and add edge to all other vertices. Here would be an formulation that could result an lower bound.

Problem6

Preprocess a rooted tree TTT where the elements in the tree TTT has weights from a semigroup (S,+)(S,+)(S,+), such that we can answer the following query in kkk semigroup operations.

Query Input: xxx and yyy where xxx is an ancestor of yyy.

Query Output: The sum of weights of any walk from xxx to yyy that lies strictly inside the subtree rooted at xxx.

Finally, here is a conjecture. Instead of using tree, we use Laminar family to simplify the wording of the conjecture.

Conjecture7

For every constant kkk, there exist a Laminar family XXX over ground set VVV of nnn vertices, such that for any graph G=(V,E)G=(V,E)G=(V,E) such that G[A]G[A]G[A] has diameter at most kkk for all A∈XA\in XA∈X, then ∣E∣=Ω(nλk(n))|E|=\Omega(n\lambda_k(n))∣E∣=Ω(nλk​(n)).

If diameter is 333, then we can show there is a Ω(nlog⁡log⁡n)\Omega(n\log \log n)Ω(nloglogn) lower bound by consider the Laminar family such that a set of size nnn gets partitioned into n\sqrt{n}n​ children each with n\sqrt{n}n​ vertices, and it is applied recursively.

The idea is at any level, say has nnn elements and n\sqrt{n}n​ subsets. A vertex is called interior if it does not connect to any vertex outside the subset, and exterior otherwise. A subset is shy if it contains a interior vertex. We show there is at least 1/5n1/5 n1/5n edges in between the subsets. If there are at least 1/2n1/2n1/2n exterior vertices, then we are done. Otherwise, there are at lease 1/2n1/2n1/2n interior vertices. We call the subsets with at least one interior vertex shy. At least n/2\sqrt{n}/2n​/2 of those subsets are shy. Consider any two distinct shy subset and pick a interior vertex from each. In order to have a distance 3 path between two interior vertices, the path has to contain two exterior vertex, one in each shy subset. But this shows if we contract all the shy subsets to one vertex, we get a clique. This means there has to be at least (n/2−1)2>1/5n(\sqrt{n}/2-1)^2>1/5n(n​/2−1)2>1/5n edges in between the subsets (for large enough nnn).
