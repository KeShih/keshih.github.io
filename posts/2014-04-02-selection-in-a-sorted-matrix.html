Selection in a sorted matrix


A matrix is sorted if every row and column are non-increasing.

A common interview problem asks for the kkkth smallest number in a sorted matrix, and usually people give an O(klog⁡k)O(k\log k)O(klogk) algorithm. Wlog we can assume all the numbers in the matrix are distinct, as we can always break the ties by factoring in the position of the number in the matrix.

There is a O(k)O(k)O(k) time solution. In fact, O(min⁡(k,m))O(\min(k, m))O(min(k,m)) if it's a n×mn\times mn×m matrix and n≤mn\leq mn≤m.

I'm frustrated that there isn't a good description of such an algorithm. The most common reference people provide is [1]. However there are a few downsides of that paper. It only works when n=mn=mn=m and it is still a bit complicated. So here I will give a short description to a modified version of the algorithm.

One can see the idea closely resemble what happens in fractional cascading, we basically squeeze the unknown values between known values, so we don't have to look at most of the unknown values.

First, we assume the matrix we care about is a n×mn\times mn×m matrix AAA. Both nnn and mmm are power of 222 and n≤mn\leq mn≤m.

Let AoA_oAo​ be the matrix we get by removing all even index columns from AAA, and add the last column.

In [1], they used Ao,oA_{o,o}Ao,o​, which is defined as only retain positions with odd coordinates, but I found it nicer to consider things by only stripping one coordinate. In particular, it gave a much nicer proof for the following result.

Definition1

r(a,A)r(a,A)r(a,A) is the number of elements in matrix AAA smaller or equal to aaa.

Theorem2

Let AAA be a sorted n×mn\times mn×m matrix, then 2(r(a,Ao)−n)≤r(a,A)≤2r(a,Ao)2(r(a,A_o) - n) \leq r(a,A)\leq 2r(a,A_o)2(r(a,Ao​)−n)≤r(a,A)≤2r(a,Ao​).

Proof

For any fixed iii, let f(i)f(i)f(i) be the largest jjj, such that Ai,j≥aA_{i,j}\geq aAi,j​≥a. r(a,A)=∑i=1nf(i)r(a,A)=\sum_{i=1}^n f(i)r(a,A)=∑i=1n​f(i), r(a,Ao)=∑i=1n⌈f(i)/2⌉≤r(a,A)/2+nr(a,A_o)=\sum_{i=1}^n \lceil f(i)/2 \rceil \leq r(a,A)/2 +nr(a,Ao​)=∑i=1n​⌈f(i)/2⌉≤r(a,A)/2+n. On the other hand ∑i=1nf(i)/2≤r(a,Ao)\sum_{i=1}^n f(i)/2 \leq r(a,A_o)∑i=1n​f(i)/2≤r(a,Ao​).

This means if we want to find an element of rank kkk in AAA, we can first find element of rank k/2+nk/2+nk/2+n and k/2k/2k/2 in AoA_oAo​, and we know the solution would be in between. The remaining operation takes O(m)O(m)O(m) time:

  1. Use a flood fill starting from the position of k/2k/2k/2th element in AoA_oAo​ and find at most 2n2n2n positions where the element of rank kkk could reside. Namely it select elements in AAA that is in between the k/2k/2k/2th element and k/2+nk/2+nk/2+nth in AoA_oAo​. We can do this because all these elements are connected in a component.
  2. While doing the flood fill, it can also find the rank of the k/2k/2k/2th element in AoA_oAo​ in the matrix AAA for no extra cost.
  3. A linear time selection algorithm on all the elements resulted from the flood fill.

This would give us a recursive algorithm if instead of just finding kkkth number, it finds the k1k_1k1​th and k2k_2k2​th number at the same time. As long as ∣k1−k2∣=O(m)|k_1-k_2|=O(m)∣k1​−k2​∣=O(m), we can find them both only with a O(m)O(m)O(m) extra time. k1k_1k1​ and k2k_2k2​ will be the upper and lower bounds respectively. Some basic algebra shows ∣k1−k2∣=O(m)|k_1-k_2|=O(m)∣k1​−k2​∣=O(m) inside each recursive step if we start with k1=k2=kk_1=k_2=kk1​=k2​=k.

Let T(n,m)T(n,m)T(n,m) be the time used when the matrix is of size n×mn\times mn×m, and n≤mn\leq mn≤m. Certainly T(1,m)=1T(1,m)=1T(1,m)=1, and the rest follow the recursive relation T(n,m)=cm+T(n,m/2)T(n,m) = cm + T(n,m/2)T(n,m)=cm+T(n,m/2) for some constant ccc.

If n>mn>mn>m, we can rotate the matrix implicitly to get the same running time.

Solving it gives us the desired running time O(m)O(m)O(m). This is also a O(k)O(k)O(k) time algorithm because we only need to consider the k×kk\times kk×k submatrix in the up-left position.

A Haskell implementation here, it requires a linear time rank selection algorithm selectRank.

Note a slightly faster algorithm also exists when nnn and mmm are very different. It was shown selecting the kkkth element can be done in O(nlog⁡m/n)O(n\log m/n)O(nlogm/n) time [2].
