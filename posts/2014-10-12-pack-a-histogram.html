Pack a histogram


Consider we have a histogram of nnn values and nnn distinct colors. The iiith bar is a rectangle of size 1×vi1\times v_i1×vi​, where viv_ivi​ is the value and has color iii. The total area of all the bars are nnn. We are interested in make a few cuts to pack it into nnn squares each one of area 111, such that each square is consist of at most 222 different colors.

Formally, we are seeking an algorithm for the following problem:

Problem1

Input: (v1,1),…,(vn,n)(v_1,1),\ldots,(v_n,n)(v1​,1),…,(vn​,n).

Output: S={s1,…,sn}S=\{s_1,\ldots,s_n\}S={s1​,…,sn​}, such that si={(x2i−1,c2i−1),(x2i,c2i)}s_i = \{(x_{2i-1},c_{2i-1}),(x_{2i},c_{2i})\}si​={(x2i−1​,c2i−1​),(x2i​,c2i​)}, x2i+x2i−1=1x_{2i}+x_{2i-1}=1x2i​+x2i−1​=1 and for all 1≤i≤n1 \leq i\leq n1≤i≤n, ∑s∈S∑(x,i)∈sx=vi. \sum_{s\in S} \sum_{(x,i)\in s} x = v_i. s∈S∑​(x,i)∈s∑​x=vi​.

Here is an algorithm that make sure this can be done in linear time. The idea is to separate the numbers into two bins. (vi,i)(v_i,i)(vi​,i) is in the large bin if vi>1v_i > 1vi​>1, and it is in the small bin otherwise. The idea is always pick two elements, one in large bin one in small bin, say (x,i)(x,i)(x,i) and (y,j)(y,j)(y,j), where x≤1<yx\leq 1< yx≤1<y. Now we create a set {(x,i),(1−x,j)}\{(x,i),(1-x,j)\}{(x,i),(1−x,j)}, and put (y−(1−x),j)(y-(1-x),j)(y−(1−x),j) back into the bins.

If we can always prove that there is an element in the small bin during the execution of the algorithm, then we find an linear time algorithm for this problem. This is clear by simple induction.

This problem come up in order to implement the alias method. I recommend a good read on this topic Darts, Dice, and Coins: Sampling from a Discrete Distribution by Keith Schwarz.
