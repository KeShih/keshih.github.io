Find the minimum of an array with a non-increasing and a non-decreasing part


1 Problem

Consider an array aaa of nnn entries from a totally ordered set. There exist a index jjj, such that a[i]≥a[i+1]a[i]\geq a[i+1]a[i]≥a[i+1] for all i<ji < ji<j, and a[i]≤a[i+1]a[i]\leq a[i+1]a[i]≤a[i+1] for all i≥ji\geq ji≥j.

How fast can we find such jjj? The worst time is O(n)O(n)O(n). When all the elements are equal, you must transverse the entire array to figure that out.

If mmm is the maximum time an element occurs in the array, then we can find an algorithm that solves this problem in O(m+log⁡n)O(m+\log n)O(m+logn) time.

2 Algorithm

The idea is to use ternary search, which works well when all values are distinct, and go for linear search once we figure there are a lot of repeated elements.

First, we present a lemma.

{Lemma} If there exist a index i<j<ki < j < ki<j<k, such that a[i]=a[j]=a[k]a[i]=a[j]=a[k]a[i]=a[j]=a[k], then at least min⁡(j−i,k−j)\min(j-i,k-j)min(j−i,k−j) positions in the array has the same value as a[i]a[i]a[i].
